# 地域ページ独自コンテンツ強化 — 設計ドキュメント

**日付:** 2026-05-25
**対象リポジトリ:** sekoukan-affiliate

---

## 背景・目的

現在の地域ページ（47都道府県 × 5工種 = 235ページ）は以下の問題を抱えている。

- `getLocalAdvice()` が5工種×3パターン=15種しかなく、複数の都道府県が同一テキストを共有している
- 「転職成功ポイント」セクションは名前を差し込むだけで全235ページが実質同文

Googleから「薄いコンテンツ」とみなされるリスクがあり、競合サイトとの差別化にも欠ける。

**目標:** `constants.ts` の既存データ（`demandLevel` / `trendKeywords` / `features` / `majorCity`）を活用し、コード側の自動生成で235ページすべてに固有のテキストを生成する。

---

## 変更スコープ

| ファイル | 変更内容 |
|---------|---------|
| `lib/types.ts` | `SuccessPoint` 型を追加 |
| `lib/constants.ts` | `ADVICE_TEMPLATES`・`getAdvicePattern` を廃止し `getLocalAdvice` を再実装。新関数 `getSuccessPoints` を追加 |
| `app/[pref]/[job_type]/page.tsx` | 「転職成功ポイント」セクションをハードコードから `getSuccessPoints()` 呼び出しに差し替え |

---

## 設計詳細

### 1. `getLocalAdvice(jobTypeId, prefId): string`

#### 現状
`ADVICE_TEMPLATES`（5工種×3固定文字列）から `prefId` のハッシュ値（% 3）でインデックスを選択。

#### 新実装
`getPrefById(prefId)` と `getJobTypeById(jobTypeId)` でデータを取得し、3つのパーツを連結して返す。

```
[需要度フレーズ] + [トレンドキーワード言及] + [工種別アドバイス]
```

**需要度フレーズ（`demandLevel` で3パターン）**

| demandLevel | フレーズ例 |
|------------|-----------|
| `high` | `{pref.name}は施工管理の求人需要が特に旺盛で、好条件の案件が多数出ています。` |
| `medium` | `{pref.name}は安定した施工管理需要があり、転職タイミングを選べば好条件の求人に巡り会えます。` |
| `low` | `{pref.name}は求人数こそ多くないものの、地域密着型エージェントを活用することで非公開求人にアクセスできます。` |

**トレンドキーワード言及（県固有）**

```
特に「{trendKeywords[0]}」「{trendKeywords[1]}」関連の案件が活発で、{jobType.fullName}経験者への需要は高まっています。
```

→ `trendKeywords` は県ごとに4件設定済みのため、先頭2件を使用。47県すべてで異なるキーワードが入る。

**工種別アドバイス（5パターン）**

| jobTypeId | アドバイス内容 |
|-----------|-------------|
| `architecture` | 1級建築施工管理技士と大手ゼネコン案件について |
| `civil` | 1級土木＋専門経験（舗装・橋梁等）と公共工事エージェントについて |
| `electrical` | 売り手市場と年収600万超えの現実性について |
| `pipe` | 省エネ設備・ZEB対応経験とCAD/BIMスキルについて |
| `landscaping` | 都市緑化需要と1級造園施工管理技士の組み合わせについて |

**結果:** 3（需要度）× 47（トレンドキーワード）× 5（工種）= 235ページすべて固有テキスト

### 2. `getSuccessPoints(jobTypeId, prefId): SuccessPoint[]`

#### 型定義（`lib/types.ts` に追加）

```typescript
export interface SuccessPoint {
  title: string;
  body: string;
}
```

#### 返却内容（3項目固定）

**ポイント1: エージェント選び（`demandLevel` で3パターン）**

| demandLevel | タイトル例 | 本文の方針 |
|------------|-----------|-----------|
| `high` | 施工管理特化エージェントに今すぐ複数登録する | 求人が動きやすい今すぐ登録を促す訴求 |
| `medium` | 地域特化型と全国型の両方に登録する | 地域密着＋幅広い選択肢のバランス訴求 |
| `low` | 全国展開型エージェントも併用して選択肢を広げる | エリアを超えた求人探しへの誘導 |

**ポイント2: 資格の活かし方（`jobType` で5パターン）**

- `jobType.license` を使い、各工種の資格取得による年収アップ効果を具体的に記述
- 工種ごとに異なる資格価値（例: 電気は特に希少性が高い、など）を反映

**ポイント3: 求人探しの戦略（`demandLevel` で3パターン）**

| demandLevel | 戦略の方向性 |
|------------|------------|
| `high` | 複数社並行応募で競争優位に立つ（条件交渉の余地を作る） |
| `medium` | 職務経歴書の数字・実績記載でライバルと差をつける |
| `low` | 希望エリアを隣県まで広げ、求人数を最大化する |

**結果:** 3（需要度）× 5（工種）= 15の組み合わせ。名前差し込みのみだった現状から実質的に差別化。

---

## 廃止するもの

- `ADVICE_TEMPLATES`（定数オブジェクト）
- `getAdvicePattern(prefId)`（ハッシュ関数）

これらは `getLocalAdvice` の新実装に吸収されるため不要になる。

---

## 影響範囲

- `getLocalAdvice` の呼び出しシグネチャ `(jobTypeId, prefId)` は変わらないため、`page.tsx` 側の呼び出し箇所の変更は不要
- `getSuccessPoints` は新関数のため、`page.tsx` の成功ポイントセクションにのみ追加変更が必要
- 静的エクスポート（SSG）のため、変更はビルド時に全ページ反映される
