# 芽の出たニッチ集中 SEO 強化 — 設計ドキュメント

**日付:** 2026-06-10
**対象リポジトリ:** sekoukan-affiliate

---

## 背景・目的

GSC 過去3ヶ月データ（2026-06-10 エクスポート）の分析結果:

- 全体: クリック 15・表示 522 — サイトは検索評価の初期段階
- 約360ページ（プログラマティック283 + 記事約77）のうち表示があるのは約40ページのみ
- ビッグKW「施工管理 転職エージェント おすすめ」は99位で当面勝ち目なし

一方で Google が「お試し表示」している芽が2つ確認できた:

| 芽 | 根拠データ |
|----|-----------|
| 造園×東京 | `/tokyo/landscaping/` 表示71回・順位19.4位（非ブランド最大）。「東京 造園施工管理 求人」31位、「新宿 造園〜」「江東区 造園〜」と区レベルクエリも発生 |
| 競合サービス指名×評判KW | 「施工管理求人ナビ 評判/口コミ」計15表示・55位、「建設転職ナビ 評判」6表示・38位、「施工管理マイナビエージェント」15表示・27位 |

施策基準ナレッジ（`施工管理サイト/転職アフィリ_施策基準ナレッジ_施工管理転職プロジェクト.md`）では
指名×評判KW を S級（CVR最高・最優先）と定義しており、GSC の芽と完全に一致する。

**目標:** 表示がつき始めた場所に集中投資し、最初の上位表示（10位以内）とアフィリエイト成約の足場を作る。

## 柱1: 造園×東京の深化（コード変更）

既存の東京固有早期リターンの仕組み（commit a46f18c で導入）を拡張する。

| 変更箇所 | 内容 |
|---------|------|
| `lib/constants.ts` `getLocalAdvice()` | tokyo×landscaping の早期リターンを区レベル情報に深化。新宿御苑・明治神宮外苑再開発・江東区臨海部の公園整備・街路樹維持管理など実在の案件・エリアを明示 |
| `lib/constants.ts` `getSuccessPoints()` | tokyo×landscaping の3項目を区レベル・案件レベルに具体化 |
| `lib/sheets.ts` `generateTokyoFAQs()` | 東京×造園の FAQ を「東京 造園施工管理 求人」の検索意図（求人の探し方・年収相場・区別の動向）に合わせて拡充 |

**制約:** 東京×造園以外の234ページに影響を与えないこと（早期リターン内の変更に限定する）。

## 柱2: S級評判記事（TSV 納品 → Sheets 貼り付け）

| 記事 | 種別 | 根拠（GSC） |
|------|------|------------|
| 施工管理求人ナビ 評判・口コミ | **新規** | 表示15回・順位55位（記事なしで拾われている） |
| マイナビエージェント×施工管理 評判 | **新規** | 表示15回・順位27位 |
| 建設転職ナビ 評判（`kensetsu-tenshoku-navi-hyouban`） | **リライト** | 表示11回・順位38位 → 情報量を競合超えに増強 |

- 構成はナレッジの収益テンプレに準拠:
  結論先出し → 基本情報 → 口コミ（良い/悪い両方） → 施工管理求人の質・量評価 → 他サービス比較 → 向き不向き → CTA 3箇所以上
- 執筆は Claude が直接行う（article-generator.ts は使わない）
- 納品形式: Google Sheets `articles` シート貼り付け用 TSV
  （列: id / slug / title / description / category / jobType / pref / body / heroImage / publishedAt / status）

### 体験談の扱い（誠実性の制約）

実際に使用していない体験談の創作はステマ・捏造リスクとなるため行わない。
記事は「編集部調査」のスタンスで、公開されている口コミ・公式データ・求人実態の調査ベースで執筆し、
実体験パートはユーザーが後から追記できるプレースホルダー構成とする。

## 柱3: 内部リンク結合

- 評判記事3本 ↔ 既存比較記事（`sekoukan-agent-osusume-2026` 等） ↔ `/tokyo/landscaping/` を相互リンク

- トップページの記事導線は既存の仕組みをそのまま利用（変更なし）
