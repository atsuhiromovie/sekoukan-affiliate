## 確定した方針（ユーザー承認済み）

1. **タイトル＝転職を主軸**に据える。
2. 網羅性の核は**完全サンプル職務経歴書（例文）**のみ追加（フォーマット雛形・工種別例・NG例は今回見送り。サンプルが構成も兼ねる）。
3. 「建設作業員／職人」クエリは**軽く拾う**（短い一節を追加）。
4. サンプル直後に**エージェント添削CTAを補強**。
5. **内部リンク追加も今回に含める**（再クロール・再評価を促す）。
6. 自己PR深掘りとAI活用は**内部リンクで住み分け**（カニバリ回避）。

## 変更内容

### A. タイトル（最重要）

現：`施工管理の職務経歴書の書き方｜現場経験をどう伝えるか`

採用案（案A'）：
```
施工管理の職務経歴書の書き方とサンプル｜転職で通る現場経験の伝え方
```
- 「転職」「サンプル」を内包しCTR改善も狙う。
- **冒頭に「施工管理の職務経歴書の書き方」を温存**：5本の既存inbound内部リンク（後述G）は `internal-links.ts` パターン3でタイトルの部分一致解決のため、この substring を残すことで**ノー編集で全リンクを維持**できる（転職を冒頭に出す案だと5本が切れるため不採用）。
- URLスラッグ（`shokumukeireki-kakikata`）は変更しない（被リンク・既存評価を温存）。
- 注意：`push-articles-to-sheets.ts` のリライト経路はC列(title)を更新しないため、入稿前にスクリプト改修が必須（プラン Task 1）。

### B. meta description 更新

「転職」「サンプル例文」を含め、新しい本文構成に整合させて書き直す。文字数は120字前後。

### C. 見出しのキーワード整合（軽微）

既存9セクションの骨格は維持。本文の質は高いため総書き換えはしない。冒頭リード文と1〜2見出しに「転職」を自然に織り込み、クエリとのズレを解消する。例：リード文に「施工管理の転職で書類選考を通すには」の文脈を一文追加。

### D.【新設】完全サンプル職務経歴書（網羅性の核）

配置：「現場経験を『数字』に翻訳する」セクションの**直後**（before/afterの具体例の次に、完成形を見せる流れ）。

構成（架空の施工管理者1名、建築系を主例に）：
- **職務要約**（3〜4行）：経験年数・主な現場規模・役割・強みを凝縮。
- **職務経歴**：現場2件。各件に請負金額・構造（S造/RC造）・延床・統括人数・協力会社数・4管理（工程/原価/品質/安全）の成果を数字入りで。
- **活かせる経験・スキル**：箇条書き。
- **保有資格**：1級/2級施工管理技士など、等級と取得年。
- **自己PR**：要点のみ短く。深掘りは [`sekoukan-jikopr-kakikata`](/articles/sekoukan-jikopr-kakikata/) へリンク送り。

Markdownのコードブロック（```）またはテーブルで提示し、内部リンク自動挿入の対象外にする（`internal-links.ts` はコードブロック・テーブル行を保護する）。

### E. 建設作業員／職人クエリを軽く拾う

「会社独自の『肩書き』を採用側の言葉に直す」または「数字に翻訳する」セクション内に、短い一節（1段落＋必要なら1〜2行の例）を追加：

- 「作業員・職人としての現場経験を、施工管理の評価軸（管理範囲・数字）に翻訳する」視点。
- 「建設作業員」「職人」「現場作業」といった語を自然に本文へ含め、該当クエリとの語彙ギャップを埋める。
- 既存の「数字に翻訳する」テーマと地続きで、焦点を二重化させない（独立セクションにはしない）。

### F. サンプル直後のエージェント添削CTA補強

サンプル提示の直後に、DIY意欲の高い読者をコンバージョン導線へ乗せる短いCTA文を置く：

- 「ここまで書けたら、第三者（施工管理特化エージェントの無料添削）で完成度を上げる」流れ。
- 既存の第三者添削セクション／アフィリエイト比較ブロック（`（※アフィリエイトリンク設置箇所）` センチネル）へ自然につなぐ。
- アフィリエイトブロックは増やさず1箇所を維持（スパム感回避）。

### G. 内部リンク（inbound／再クロール促進）

調査の結果、本記事には**既に5本のinbound内部リンクが存在**する（複数内定・自己PR・転職理由・同業種転職・バレずに）。いずれも `（内部リンク：施工管理の職務経歴書の書き方）` 形式で、`internal-links.ts` パターン3が**タイトルの部分一致**で解決している。

したがって今回の方針は「新規追加」ではなく「**既存5本を壊さず温存する**」こと：
- 案A'はタイトル冒頭に「施工管理の職務経歴書の書き方」を残すため、5本は**兄弟記事を一切編集せず**自動的に生き続ける。
- 本文・metaの更新による再入稿そのものが、本記事の再クロール・再評価のトリガーになる。
- 検証：公開後に兄弟記事1本（jikopr）をライブ取得し、本記事へのリンクがテキスト化していない（生きている）ことを確認（プラン Task 8 Step4）。

新規inbound追加は今回スコープ外（必要なら低コストの follow-up）。
